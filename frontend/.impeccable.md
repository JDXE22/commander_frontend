# Impeccable Design Context

## Project: Commander

### Users
Tech-savvy general users who need to interact with the Commander API. They value efficiency but also require visual clarity and onboarding.

### Brand Personality
**Cyber-Terminal**: Technical, efficient, and futuristic. The aesthetic is inspired by classic terminals and command-line interfaces but adapted for a modern web experience.
- **Tone**: Professional, precise, slightly edgy.
- **Personality**: Cyberpunk-industrial, functional, high-tech.

### Aesthetic Direction
A "terminal-punk" interface that blends monospaced typography with modern UI patterns.
- **Theme**: Support for both Light and Dark modes.
  - **Dark Mode**: Classic high-contrast green-on-black terminal vibe with glowing accents.
  - **Light Mode**: A "Paper-Terminal" or "Solarized Light" aesthetic—clean, readable, but retaining the monospaced character and technical layout.
- **Typography**: JetBrains Mono for technical data and labels; Inter for body text and general interface elements.
- **Key Visuals**: Terminal-green accents, subtle scanlines or noise textures (optional/delight), clear hierarchy of command history, and expressive motion for command execution.

### Design Principles
1. **Clarity over Chaos**: The terminal vibe should never compromise the user's ability to find and execute commands.
2. **Keyboard-First, Mouse-Friendly**: Optimize for shortcuts but ensure every action is clear for mouse users.
3. **Tactile Feedback**: Use motion and toasts (`sileo`) to make the interface feel "alive" and responsive to user input.
4. **Contextual Onboarding**: Guide the general tech-savvy user through the command discovery process without overwhelming them with "pro" jargon.

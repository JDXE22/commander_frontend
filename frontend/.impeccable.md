## Design Context

### Users
Working professionals across all fields who need quick access to text templates — not just developers. The interface must be approachable enough for beginners landing here for the first time while remaining efficient for power users who rely on it daily. Context: fast retrieval under time pressure (composing emails, filling forms, responding to messages).

### Brand Personality
**Efficient, clean, approachable.** Commander should feel like a sharp tool that anyone can pick up — no learning curve, no intimidation. Terminal aesthetics are a design choice, not a gatekeeping signal.

### Aesthetic Direction
- Dark mode with terminal-green accent (`#22c55e`)
- Olive-green sidebar creating a nature-meets-utility feel
- Inter for UI, JetBrains Mono for command/code elements
- Card-based layouts with generous spacing
- Minimal decoration — let content and actions breathe

### Design Principles
1. **Speed over ceremony** — Every interaction should feel instant. Minimize steps between intent and result.
2. **Obvious over clever** — If a beginner can't figure it out in 3 seconds, it's too complex. Labels, affordances, and hierarchy must be self-explanatory.
3. **Quiet confidence** — The interface should feel capable without being loud. No unnecessary flourishes, no visual noise.
4. **Consistent feedback** — Every action gets clear, immediate confirmation. Toasts, state changes, and transitions should reinforce what just happened.
5. **Approachable terminal** — Terminal aesthetics for character, not gatekeeping. The "$" prompt and monospace fonts are a vibe, not a prerequisite.
